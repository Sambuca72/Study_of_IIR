## Задача №3

| p   | omp parallel for                    | omp parallel                        |
| --- | ----------------------------------- | ----------------------------------- |
| 2   | $T_{p}$ = 45.3s \| $S_{p}$ = 1.98x  | $T_{p}$ = 43s    \| $S_{p}$ = 2x    |
| 4   | $T_{p}$ = 22.6s \| $S_{p}$ = 3.97x  | $T_{p}$ = 22.2s \| $S_{p}$ = 4x     |
| 7   | $T_{p}$ = 12.8s \| $S_{p}$ = 7x     | $T_{p}$ = 12.9s \| $S_{p}$ = 7x     |
| 8   | $T_{p}$ = 11s    \| $S_{p}$ = 8.1x  | $T_{p}$ = 11.2s \| $S_{p}$ = 8x     |
| 16  | $T_{p}$ = 5.9s   \| $S_{p}$ = 15.1x | $T_{p}$ = 6.7s   \| $S_{p}$ = 13.4x |
| 20  | $T_{p}$ = 5.2s   \| $S_{p}$ = 17.2x | $T_{p}$ = 5.4s   \| $S_{p}$ = 16.4x |
| 40  | $T_{p}$= 3.9s    \| $S_{p}$ = 23.5x | $T_{p}$ = 5.3s   \| $S_{p}$ = 17x   |

<img width="1966" height="1033" alt="Снимок экрана 2026-04-13 220439" src="https://github.com/user-attachments/assets/561d07a2-aa6d-476a-abd7-5bf7e7620f6d" />

Сравнение директив `omp parallel for` и `omp parallel` показывает:

- **До 20 потоков:** Оба метода идут практически вровень, обеспечивая стабильное ускорение.
- **На 40 потоках:** Комбинированная директива `omp parallel for` выигрывает с результатом **23.5x** против **17x** у ручного разделения. Это подтверждает, что встроенный планировщик OpenMP эффективнее распределяет нагрузку на больших мощностях.
- **Dynamic:** показал наилучшие результаты в среднем, особенно на 40 и 80 потоках, где он дал максимальное ускорение ($\approx 15.51$).

  **Вывод:**
- **Dynamic** или **pragma parallel** — лучший вариант для масштабируемости на больших потоках.
- **Pragma for** — лучший вариант до 40 потоков.
